# 4.函数

## 4.1. 什么是函数

函数是程序中独立的代码块，用于执行特定的任务。

函数可以提高代码的可读性，可复用性和模块化

1. **代码复用**：避免重复编写相同代码。
2. **逻辑分离**：将程序划分成更小的部分，便于调试和维护。
3. **提高可读性**：函数名通常描述了其功能，易于理解。



## 4.2. 函数的定义和调用

### 4.2.1 基本语法

```c++
int foo() { return 0 };
```

- **返回类型**：函数返回值的数据类型，例如 `int`、`void` 等。
- **函数名**：标识函数的名称。
- **参数列表**：传递给函数的输入数据，可以为空。
- **函数体**：函数要执行的代码。
- **return**：用于返回函数的结果，如果返回类型是 `void`，则可以省略 `return`。

## 4.2.2. 基本函数

#### **无参无返回值的函数**

```c++
#include <iostream>
using namespace std;

void sayHello() {
    cout << "Hello, World!" << endl;
}

int main() {
    sayHello(); // 调用函数
    return 0;
}
```

#### **有参有返回值的函数**

```c++
#include <iostream>
using namespace std;

int add(int a, int b) { // 定义一个接收两个参数的函数
    return a + b; // 返回两数之和
}

int main() {
    int result = add(5, 3); // 调用 add 函数，并传入参数
    cout << "Result = " << result << endl;
    return 0;
}
```

## 4.3. 函数参数传递方式

### 4.3.1. 按值传递

- 传递参数的副本，函数内修改不会影响原来的值
- **值传递** 的本质是 **拷贝**，当你把一个变量传递给函数时，函数内部接收到的是 **原变量的副本**。任何对副本的修改都不会影响原变量。

##### **底层逻辑**

- 在值传递的过程中，函数参数在内存中会创建一块新的存储空间
  - 用于存储传递过来的变量的 **拷贝**。

- 这块新的空间和原始变量 **没有任何关系**，它们独立存在。
- 修改函数内部的参数，只是修改这个拷贝的值，而不会影响原变量。

```c++
#incloud <iostream>
using namespace std;

void foo(int x) {  // 这里 x 是一个拷贝
  x = 10; // 修改的是 x 的副本
};

int main() {
  int num = 5;
  foo(num); // 在函数中没有改变num的值
  cout << num << endl; // num = 5
  return 0
}
```

### 4.3.2. 按引用传递

- 传递参数的引用, 函数内修改值, 会影响到外层作用域的值\
- **引用传递** 的本质是 **别名**。
- 当你使用引用传递时，函数参数会成为原始变量的一个别名，引用和原变量指向同一个内存空间。

##### **底层逻辑**

- 引用并不会创建新的存储空间，而是将函数参数和原变量绑定起来。
- 修改引用，就相当于修改原变量，因为它们在内存中是同一块空间。

```c++
void changeValue(int &x) { // x 是 num 的引用，是 num 的别名
  // 获取变量的内存地址, 这种修改直接改内存地址中的值
  x = 10;
};

int main() {
  int num = 5;
  changeValue(num);
  cout << num << endl; // num = 10
  return 0
}
```

### 4.3.3. 按指针传递

根据内存地址, 重新开辟一块空间, 指针本身也占据空间

- **指针传递** 的本质是 **传递内存地址**，通过指针间接访问原始变量的内存空间，从而修改变量的值。 

##### **底层逻辑**

- 通过传递指针，函数接收的是变量的 **内存地址**。
- 指针可以通过解引用操作 `*` 访问该地址上的数据，从而修改原变量。

```c++
#include <iostream>
using namespace std;

void modifyValue(int *x) { // x 是指向 num 的指针
    *x = 10; // 解引用，修改 num 的值
}

int main() {
    int num = 5;
    modifyValue(&num); // 传递 num 的地址
    cout << num << endl; // 输出 10，原值被修改
    return 0;
}
```

### 4.3.4. 引用和指针的区别

| 特性           | **引用**                     | **指针**                         |
| -------------- | ---------------------------- | -------------------------------- |
| 是否可以为空   | 不可以为空，必须绑定有效变量 | 可以为空，指针可以指向 `nullptr` |
| 是否可重新赋值 | 不可重新绑定，绑定后就固定   | 可以重新指向其他地址             |
| 语法简洁性     | 使用时无需解引用 `*`，较简单 | 需要显式解引用 `*` 和取地址 `&`  |
| 内存是否独立   | 和原变量共享内存空间         | 指针本身占据一块内存             |
| 用途           | 更适合传递已知有效变量的引用 | 更灵活，可以动态操作内存         |

### **4.3.5. 同时存在的意义**

#### **值传递的意义：**

- 适用于传递小型数据（如 `int`、`char`）时，修改参数不会影响原数据。
- 简单直接，避免不必要的副作用。

#### **引用传递的意义：**

- 适用于需要在函数中修改原变量的场景。
- 语法简洁，传参无需解引用。

#### **指针传递的意义：**

- 适用于复杂的数据结构，如动态数组、链表等。
- 可以传递空值，指针更灵活。
- 适用于需要动态内存分配的情况。

### 4.3.6**总结**

1. **值传递**：传递变量的 **副本**，修改不影响原值。
2. **引用传递**：传递变量的 **别名**，修改直接影响原值。
3. **指针传递**：传递变量的 **地址**，通过解引用修改原值。

**引用和指针的区别：**

- 引用是原变量的别名，**不可为空且绑定后不可更改**。
- 指针更灵活，可以指向不同的地址，可以为 `nullptr`。
